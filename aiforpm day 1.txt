activity 3: design a multi-variable prompt for agile user story generation

bad:
write some user stories for a login page.

better:
[role] act as an expert agile business analyst specializing in financial technology platform.

[context] PM Ronaldo is managing the delivery of a new secure mobile banking portal. Senator Robin is the executive sponsor who demands strict adherance to security compliance.

[task] draft 3 comprehensive User Stories for the 'Two-Factor Authentication (2FA)' feature configuration.

[constraints] 
1. each story must precisely follow the format: 'As a [user role], I want to [action] so that [benefit].'
2. every user story must include at least 3 distinct Acceptance Criteria using Given-When-Then behavioral framework.
3. ensure one user story specifically handles an error path (e.g., wrong OTP entered)

[formatting] output your response using markdown headers for each user story with the acceptance criteria listed as sub-bullet points.

activity 4 : chained-prompting method for escalation email workflows

multi-stage stakeholder communication plan without starting a new chay session

1. root analysis
act as a senior project manager. our project, led by PM Rod, has encountered a 3-week delay because our primary vendor failed to deliver the security audit clearance documentation. the project sponsor is mr.escudero.
First, analyze this scenario and list 3 immediate downstream impacts this delay will cause on subsequent project phases. 

2. direct escalation
based on the impacts, draft a formal escalation email from PM Rod to the vendor's account director, requesting am urgent remediation meeting. The email must be firm, objective, specify a deadline for response within 24 hours, and mention that sponsor mr.escudero has been briefed on the issue.

3. executive summary
now, condense the entire situation into a 3-bullet executive summary tailored for an executive dashboard read by sponsor mr. escudero. Focus on status, financial impact (estimate a realistic penalty cost), and next steps. Keep it incredibly concise.
